Open Drupal Bootcamp

What will I learn about?

You will learn about the following within practical exercises:

  • What Content Management Systems, Open Source, and Drupal are and the advantages of each.
  • The basics of HTML
  • How to install and configure Drupal and how to manage content.
  • The fundamental concepts of Drupal including: Nodes, Taxonomy, Menus, Blocks and Themes.
  • Some of the most widely used Drupal modules, including Views & Webform.
  • An introduction to web servers and the underlying technologies needed to host a Drupal website.
  • Set up a local environment, and install Drupal locally ready for development work
  • Install some tools to help with local development.
  • Get stuck into the fundamental languages that underpin the web: HTML & CSS.
  • Set up a rich text editor and learn about text formats.
  • Use Drupal’s powerful user management features and contributed modules add interactive functionality.

What will I have to show after I finish?

  • A fully working website built with Drupal, hosted online and publicly accessible.
  • You website will have a blog with categorised posts, with as many pages and menu links as you like and custom ‘blocks’ of content where ever you would like to add them.
  • The powerful content management capabilities of Drupal at your fingertips.
  • Social media integration and a custom homepage.
  • A website you can grow, and the knowledge and capability to modify the structure and content of your website with just a few clicks.

3. Content Management Systems

Why a CMS?

HTML was originally designed as a means of sharing and linking together individual documents. Plain HTML is still adequate for simple documents, but it is very limiting as it means a new, separate document has to be created for each new page or article. By keeping the content of a website in a database, and using a pre-built system to manage and display this content, it is easy to build and run large, flexible and maintainable websites.

A website using Drupal as its CMS has four main components:

The details of how these components interact and how they are configured isn’t that important at this stage; the benefit of a pre-built platform that we are going to use is that this is taken care of to begin with. Other CMSs work similarly, but with different components, such as .NET/Ruby/Java/Javascript for the backend language, or PostgreSQL/MongoDB for the database.

There are lots of choices when it comes to CMSs; we could build our own, or we could use one that has already been built. Using an existing CMS means we can very powerful systems without having to do nearly as much work. The code has also been tested thoroughly and we can get access to a whole community which offers loads of add-ons and support. Other developers who come along after us to update sites will also be familiar with the structure.

Different CMSs can be good for different things, take a look at some of the different CMSs out there:

  • Wordpress
  • Joomla
  • Drupal

6. Manage your site

A. Nodes

A ‘node’ is a piece of content on a Drupal site, if you’ve set your site up and created a page or an Article then you will have created a node already! To go over this process again, and to learn more about some of the options you have when creating nodes:

  • Watch: Creating and editing nodes
  • Watch: Node administration

B. Menus

Menus are a fundamental part of most websites. Drupal comes with a powerful system of managing menus out of the box.

  • Watch: Menus

C. Create some content

Now we’re going to add some nodes with menu links, out of the box Drupal comes with two different ‘Node types’: Articles, and Basic Pages:

Create some Articles

  1. Add two new Articles to your site.
  2. Make one of them unpublished
  3. Then publish it from the content administration page

Add some pages with menu links

Now, we’ll enable the Navigation menu in the Basic Page content type, create a page with a menu link in the Navigation menu, and then create another page as a child of the first.

  1. Go to Structure -> Content Types and click on ‘edit’ for the Basic Page content type.
  2. Enable ‘Navigation’ in ‘Available menus’
  3. Now, add a  basic page  and call it something like About me, under ‘Menu Settings’ select ‘Provide a menu link’, and select <Navigation> as the Parent item.
  4. Now, add another Basic page, this time call the page ‘Experience’. Add a menu link and select the ‘About me’ page you just added as the Parent item.
  5. To manage menus within your site, go to ‘Structure->Menus’ - can you think of a better structure for the menus in your site? Go ahead and move menu items around.

D. Taxonomy

When we want to categorise content we can use the ‘Taxonomy’ system that Drupal comes with.

  1. Read: Drupal.org Taxonomy documentation
  1. Set up a new Taxonomy

As we add more and more Articles to our site, it could be useful to ‘group’ them together in similar subjects. We can do this with Taxonomy.

Add a new vocabulary

  1. Add a new taxonomy vocabulary, call it ‘Areas of interest’
  2. Add a few terms within this vocabulary which will be the main areas of your site (e.g ‘Apprenticeship posts’, ‘Tech’, ‘Cool sites’)

Add a term reference field

  1. Go to Structure->Content Types and Manage Fields for the Article Content Type
  2. Add a new field. Call it ‘Area of interest’ and select Term reference as the Field Type. Chose Select List for the widget.
  3. Click ‘Save’
  4. Now select ‘Areas of interest’ on the field settings page for the Vocabulary and click save.
  5. On the next page we can leave all of the default values as they are so click save.

Now, when we create a new Article we’ll have the option to categorise by Area of interest

Categorise our content

  1. Create a few more dummy blog posts and pages (you can use a lipsum generator for dummy content) and categorise them by selecting the ‘Area of interest’ for each of them
  2. Now, when viewing the blog posts you can click on an Area of interest and see other blog posts with the same Taxonomy terms

    13. Rebuild & Import

    Intro

    Now you’ve got a local development environment with Drupal up and running, we can install some common modules to begin some more in-depth development.

    Install Modules

    Now we have a local setup we can install modules by copying them directly in to our local codebase.

    • Read: Installing contributed modules
    • Download and install Views, CTools & Webform modules
    • Install a theme if you like

    Export & Import

    Now we’re going to export data from our cloud site, and Import to our local site. There are various ways we can go about this:

    Manual Import & Export

    We could simply re-create content on our local site, in the same way our cloud site is setup. Simply copy & pasting content is a quick and simple method that should always work, although it can be time consuming. Be careful to check you don’t miss anything!

    Export & Import Nodes

    Generating PHP code to Export & Import is less error prone and can be quicker.

    • Install the Node Export module on your cloud site
    • Go through each node individually and export, nodes will be exported in PHP
    • On your local site, import each node by copying & pasting the PHP code.
    • We could use the Views Bulk Operations module to export a number of nodes at a time, however this requires more advanced configuration

    Import & Export Views

    Views comes with Export & Import functionality.

    • Navigate to a view you’d like to export from your dev site and select ‘export’
    • Copy the view export code
    • On your local site, go to Structure->Views and click Import
    • Paste the code in and import the view

    Export & Import your database

    We can take a snapshot of our cloud site’s database, and overwrite our local database. This method can be quick, but is more prone to errors and breaking your site if anything goes wrong.

    • First make sure your local codebase is synchronised with your cloud site’s. You’ll need to check the same version of Drupal and all of the same modules & themes are available locally.
    • We could use SFTP to copy all of the cloud site’s files to our local installation. Be careful to ensure that your local settings.php file isn’t overwritten as it contains the details to connect to your local database.
    • Now we can use a module called backup & migrate to export our cloud site’s database, and import it to our local site. We could also import the database using PhpMyAdmin, first deleting all tables in our local database and then importing the backup generated from Backup & Migrate.

    13. Improve the editing experience

    Improve The Content Editing Experience

    Intro

    To make life easier for our content editors, we can install an RTE (Rich Text Editor), sometimes known as a WYSIWYG (What You See Is What You Get) Editor.

    Combined with Drupal's input (text) formats an RTE will allow us a good deal of control over what users can input into our site and at the same time give them (and us) an easier way of adding text content.

    Set up CKEditor

    • Install the CKEditor module
    • The CKEditor module requires the CKEditor Javascript library, you’ll need to download this and copy it into the correct place
    • We’ll also need to configure the CKEditor module, enabling buttons and making sure it’s being applied to the correct text formats.

    Set up a new text format

    Text formats in Drupal allow us to control what users are allowed to add into text fields, and how what they add is altered. They can let us do things like change URLs into links automatically, or strip out HTML code if we don’t want users to be able to add this in. It’s usually good practice to strip out unwanted code so we can control the way our site looks more closely.

    • Create a new text format called “Rich Text”
    • Set this as the default text format, and enable CKEditor for this format

    14. Create your own Sub-theme

    Intro

    It is possible to 'hack' themes - changeing or adding code to the template files to make them look and behave how you would like them to. However, developing in this way creates a fork of the original theme which can no longer accept updates from the team that created it. Drupal provides us with a getaround enabling us to create our own themes based on other themes without the risks and responsibilities that come with hacking or forking existing projects.

    We create a sub-theme, one that inherits all the files and properties of the original theme but can safely contain our individual overrides for style and functionality.

    • https://www.drupal.org/node/1978010
